## Panel Registry + Focus Service (Tier 4 — App.tsx/PanelRenderer Refactor)

Three extractions to reduce coupling in App.tsx and PanelRenderer.tsx.

### Focus Service (`src/lib/focus-service.ts`)

Extracted DOM-query-based focus management from App.tsx into a dedicated module.

- `focusTabPanel(tabId)` — queries `[data-tab-panel-id]`, finds `[data-tab-focus-target]` or first focusable
- `focusSidebarPrimary()` — targets sidebar first focusable
- `restorePrimaryWorkspaceFocus(activeTabId, ideRoot?)` — falls back to active tab panel or IDE root
- `sidebarHasFocus()` — checks if active element is inside `.sidebar-wrap`

### Panel Registry (`src/lib/panel-registry.ts`)

Replaced PanelRenderer's 13-branch if/else chain with a declarative registry.

- `PanelDefinition` type: `{ render(tab, isActive, deps) → JSX.Element }`
- `PanelDeps` interface: shared deps for all panels (workspaceRoot, settings, handleFileSelect, etc.)
- `FileTabDeps` interface: editor-specific deps (engineMap, searchMatches, diffHunks, etc.)
- `registerPanel(type, def)` / `getPanel(type)` API

### Panel Definitions (`src/lib/panel-definitions.tsx`)

12 non-file panel types registered as clean one-liners: terminal, settings, git, extensions, search-results, problems, manual, debug, explorer, image, surface.

PanelRenderer's `createPanelElement` is now: check registry → if found, render via definition → else fall through to `renderFileTab()` (which has unique breadcrumb/blog/engine concerns).

## GPU Text + Display Row Memoization (Tier 2)

- **GPU text renderer enabled** — `webgl-text.ts` (WebGL2 glyph atlas + instanced rendering) wired into CanvasEditor. All text drawn in a single instanced draw call. Falls back to Canvas 2D if WebGL2 unavailable.
- **Display row memoization** — `getDisplayRows()` caches `computeDisplayRows()` results across frames. Only recomputed when lines, charW, editorWidth, wordWrap, gutterW, or foldedLines change. Eliminates O(n) per frame for cursor blink, scroll, overlay changes.

## Shell-Word Parsing (Tier 3)

Added `parse_shell_words()` in `src-tauri/src/extensions/runtime.rs`. Handles double quotes, single quotes, and backslash escapes. `git commit -m "fix bug"` now correctly splits into `["git", "commit", "-m", "fix bug"]` instead of splitting on every space.
