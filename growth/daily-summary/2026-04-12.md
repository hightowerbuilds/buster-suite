## Panel Registry + Focus Service (Tier 4 — App.tsx/PanelRenderer Refactor)

Three extractions to reduce coupling in App.tsx and PanelRenderer.tsx.

### Focus Service (`src/lib/focus-service.ts`)

Extracted DOM-query-based focus management from App.tsx into a dedicated module.

- `focusTabPanel(tabId)` — queries `[data-tab-panel-id]`, finds `[data-tab-focus-target]` or first focusable
- `focusSidebarPrimary()` — targets sidebar first focusable
- `restorePrimaryWorkspaceFocus(activeTabId, ideRoot?)` — falls back to active tab panel or IDE root
- `sidebarHasFocus()` — checks if active element is inside `.sidebar-wrap`

### Panel Registry (`src/lib/panel-registry.ts`)

Replaced PanelRenderer's 13-branch if/else chain with a declarative registry.

- `PanelDefinition` type: `{ render(tab, isActive, deps) → JSX.Element }`
- `PanelDeps` interface: shared deps for all panels (workspaceRoot, settings, handleFileSelect, etc.)
- `FileTabDeps` interface: editor-specific deps (engineMap, searchMatches, diffHunks, etc.)
- `registerPanel(type, def)` / `getPanel(type)` API

### Panel Definitions (`src/lib/panel-definitions.tsx`)

12 non-file panel types registered as clean one-liners: terminal, settings, git, extensions, search-results, problems, manual, debug, explorer, image, surface.

PanelRenderer's `createPanelElement` is now: check registry → if found, render via definition → else fall through to `renderFileTab()` (which has unique breadcrumb/blog/engine concerns).

## GPU Text + Display Row Memoization (Tier 2)

- **GPU text renderer enabled** — `webgl-text.ts` (WebGL2 glyph atlas + instanced rendering) wired into CanvasEditor. All text drawn in a single instanced draw call. Falls back to Canvas 2D if WebGL2 unavailable.

### GitHub References Removed

### Terminal — 8 Features in One Pass

Deep review of the terminal system (~3,400 lines across 13 files) identified 6 "accentuate" items + high/medium impact gaps. Executed a 5-hour plan in a single Rust-then-frontend sweep.

**Rust backend (`mod.rs` + `commands/terminal.rs`):**
- `TermCell` gained `width` (0/1/2 for CJK), `strikethrough`, `faint` fields
- `TermScreenDelta` gained `title` (OSC 2), `bell` (BEL count), `alt_screen` fields
- Reader thread tracks bell count, title changes, alt-screen state across deltas
- New `on_pty_restart` callback emits `terminal-pty-restart` event on crash recovery
- `PtyRestartEvent` struct added for frontend consumption

**Frontend (`CanvasTerminal.tsx`):**
1. **CJK double-width** — skip `width===0` continuation cells, render wide chars across `cw*2` pixels, width-aware backgrounds
2. **Alt-screen scrollback isolation** — dual buffers (`scrollbackNormal`/`scrollbackAlt`), swap on `alt_screen` transitions, discard alt buffer on exit
3. **OSC 2 tab titles** — `delta.title` propagated via new `onTitleChange` prop → `BusterProvider.handleTermTitleChange` → tab name update
4. **Bell visual flash** — 150ms translucent overlay on `delta.bell`, self-clearing via rAF
5. **Crash recovery toasts** — listens for `terminal-pty-restart` (info) and `terminal-pty-error` (error) events, shows via `showToast()`
6. **Strikethrough rendering** — horizontal line through cell midpoint (ready when vt100 exposes SGR 9)
7. **Faint rendering** — 50% alpha on foreground color (ready when vt100 exposes SGR 2)
8. **Terminal search** — `Cmd+F` opens HTML overlay with match highlighting, `Enter`/`Shift+Enter` to navigate, `Escape` to close. Searches scrollback + visible cells.

**Supporting changes:**
- `CanvasSurface.tsx` gained `searchOverlay` prop for terminal search UI
- `PanelDeps` interface gained `handleTermTitleChange`
- `buster-context.ts` and `App.tsx` wired through the new action

### Tab Bar — Close Button Fix + Double-Click Rename

**Close button fix:** Hit regions were pushed in wrong order — close region added before tab region, but `canvas-chrome` walks regions in reverse (last wins). Close button clicks were being swallowed by the tab region underneath. Swapped push order so close region takes priority.

**Double-click rename:** Added `onDblClick` support to `HitRegion` interface and `CanvasChrome` component. Double-clicking any tab opens an inline `<input>` overlay positioned exactly over the tab name (using measured `nameX`/`nameW` from paint). Enter commits, Escape cancels, blur commits. `onRename` prop on `CanvasTabBar` wired through `App.tsx`.

**Overlay alignment fix:** Initial implementation used a rough `PAD + 20` offset. Refined to store exact name geometry (`nameX`, `nameW`) in `tabRects` during paint, so the input aligns pixel-perfectly with the canvas-rendered text. Full bar height, no border, accent underline, palette colors.

### Delete Key Cursor Bug Fix

**`engine.ts deleteForward()`** — single-cursor no-selection path called `setLines(ls)` without `batch()`, leaving cursor position unprotected from intermediate reactive updates. Fixed to `batch(() => { setLines(ls); setCursor(p); })` matching `backspace()`.

**Hidden textarea double-fire** — on some Tauri WebView implementations, `e.preventDefault()` in keydown doesn't fully suppress textarea input events. Added explicit `hiddenInput.value = ""` on every special key (Backspace, Delete, Alt+Backspace) in both editor and terminal keydown handlers.

## Built-in Browser Integration (Late Night Session)

Converted the embedded browser from a manually-installed WASM extension into a first-class built-in feature. The browser WASM module (78KB) now ships bundled inside the app — users open it via `Cmd+Shift+B` or the command palette with zero setup.

### Backend (`browser_module.rs` + `commands/browser_module.rs`)

- **`BrowserModule`**: Loads `browser.wasm` from Tauri's bundled resources directory. Constructs a synthetic `ExtensionManifest` with `render_surface`, `browser_control`, and `notifications` capabilities. Reuses the extension system's `link_host_functions` and `ExtensionState` unchanged — no refactoring of the extension system required.
- **Lazy initialization**: `Mutex<Option<BrowserModule>>` in Tauri managed state. Only instantiated on first `Cmd+Shift+B`.
- **10 Tauri commands**: `browser_module_launch`, `navigate`, `refresh`, `poll`, `on_click`, `on_key`, `on_resize`, `on_visibility`, `on_mouse_move`, `close`.
- Made `link_host_functions` `pub(crate)` for reuse. Wrapped `SurfaceManager` in `Arc` for shared ownership.

### Frontend (`BrowserSurface.tsx` + wiring)

- **`BrowserSurface.tsx`** (234 lines): Adapted from `DisplayListSurface`. Routes input events directly through `browserModule*` IPC (not `extCall`). 500ms poll interval for devtools data. Calls `browserModuleClose()` on unmount.
- **`"browser"` tab type**: Registered in `tab-types.ts` and `panel-definitions.tsx`.
- **Surface-event interception**: Listener in `BusterProvider` intercepts `"__builtin_browser"` extension_id to create `"browser"` tabs instead of `"surface"` tabs.
- **`Cmd+Shift+B`**: Keyboard shortcut + "Open Browser" command palette entry.
- **Tab close**: Browser tab close calls `browserModuleClose()` (separate cleanup from `extUnload`).

### Browser UX Polish

- **URL bar prefills with `localhost://`** — users immediately see the pattern and type a port number.
- **URL bar starts focused on launch** — Backspace/Delete work immediately. Previously `url_focused` started `false`, causing the key handler to silently ignore non-printable keys until the user clicked the URL bar.
- **Text renders in primary color** (not greyed out) since input is non-empty and focused.
