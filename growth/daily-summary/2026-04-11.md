## Copy/Paste Fixed

Clipboard operations in the Tauri dev window were silently failing because `navigator.clipboard.readText()` requires permissions that the dev webview doesn't always grant.

- Removed Cmd+C/X/V handlers from `handleKeyDown` (they called `e.preventDefault()` which blocked native clipboard events)
- Added `onCopy` and `onCut` handlers to the hidden textarea using `e.clipboardData.setData()` (synchronous, no permissions needed)
- The existing `onPaste` handler already used `e.clipboardData.getData()` but was never reached because keydown prevented default

## buster-syntax Integration (Tier 2 — Incremental Tree-Sitter)

Integrated the `buster-syntax` crate for persistent per-document parse trees with viewport-scoped highlighting. Replaces the stateless full-reparse-on-every-request approach.

**Rust backend (3 files):**

- **`TreeSitterProvider`** (`syntax/mod.rs`) — implements `ParseProvider` by wrapping tree-sitter-highlight events. Converts byte-offset highlight events to per-line `HighlightSpan{line, start_col, end_col, kind: TokenKind}`. Maps all 27 `HIGHLIGHT_NAMES` to `TokenKind` variants (Keyword, Function, String, Comment, etc.) via `highlight_index_to_token_kind()`. Uses binary search for line lookup.
- **Document management** — `SyntaxService` gains `documents: RwLock<HashMap<String, Mutex<DocumentTree>>>` with `open_document()` (creates DocumentTree with TreeSitterProvider, does initial reparse), `close_document()`, `edit_document()` (apply_edit + reparse), `highlight_viewport()`.
- **Tauri commands** (`commands/syntax.rs`) — `syntax_open`, `syntax_close`, `syntax_edit` for lifecycle. `highlight_code` updated: accepts `start_line`/`end_line`, tries document-backed viewport path first, falls back to stateless parse. All registered in `lib.rs`.

**TypeScript frontend (3 files):**

- **`ts-highlighter.ts`** — New `HighlightSpan` interface: `{line, start_col, end_col, kind}`. `KIND_TO_SYNTAX` maps TokenKind names to theme syntax keys. `spansToLineTokens()` simplified: spans are already per-line, only does byte→char conversion per line (no whole-file offset mapping). Added `syntaxOpen()`, `syntaxClose()`, `syntaxEdit()` IPC wrappers. `requestHighlights()` now accepts viewport range.
- **`CanvasEditor.tsx`** — Calls `syntaxOpen(fp, text)` on mount, `syntaxClose(fp)` on cleanup. `refreshHighlights()` computes viewport from scrollTop/canvasHeight/lineHeight with ±5 line buffer. Cache invalidation via `highlightDirty` flag (set by `clearHighlightCache()` on every edit).
- **`ts-highlighter.test.ts`** — Rewritten for new per-line span format.

**Fallback preserved:** If `syntax_open` wasn't called, `highlight_code` creates a temp DocumentTree on the fly. No breakage for non-lifecycle paths.
