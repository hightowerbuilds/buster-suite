### Crash: Closing Cmd+N Untitled Tab

Reproduced user-reported scenario:

1. Open app.
2. Press `Cmd+N` to create an untitled file tab.
3. Click the tab close button.
4. App exits and macOS reports `CFRelease() called with NULL`.

Crash report pointed at:

- `notify::fsevent::FsEventWatcher::remove_path`
- `buster_lib::watcher::FileWatcher::unwatch`
- `commands::file::unwatch_file`

Root cause:

- Untitled tabs are file tabs with `path: ""`.
- The tab close flow always called `unwatchFile(tab.path)`.
- Passing an empty path into macOS FSEvents through `notify.unwatch` could crash inside CoreFoundation.

### Fix

Frontend:

- `src/lib/actions-tabs.ts`
- Guarded the unwatch IPC call so untitled tabs do not call `unwatchFile("")`.

Backend:

- `src-tauri/src/watcher.rs`
- Added `watched_paths` tracking so the Rust watcher only calls `notify.unwatch` for paths it actually registered.
- Made empty, invalid-relative, or never-watched paths no-op on unwatch.
- Added duplicate-watch suppression.
- Cleans debounce and suppression state after successful unwatch.

## Session 2: Editor Language Snippet Foundation

Started the editor-specific work by moving HTML boilerplate expansion out of a hard-coded input branch and into a small language-aware snippet layer.

### Goal

Support the common IDE workflow where an HTML file can expand `!` into a full HTML document skeleton, including the unsaved-file workflow where a user creates a new tab and renames it to something like `index.html` before saving.

### Implementation

Created `src/editor/language-snippets.ts`:

- `inferLanguageId(languagePath)` maps file paths or tab names to language IDs.
- HTML, JavaScript, TypeScript, CSS, JSON, Rust, Python, and Go extensions are recognized as a foundation for later language-specific snippets.
- `expandTypedLanguageSnippet()` supports immediate typed-trigger expansion.
- `expandLanguageSnippetBeforeCursor()` supports trigger-then-Tab expansion.
- HTML `!` expands to:
  - `<!DOCTYPE html>`
  - `<html lang="en">`
  - UTF-8 charset meta
  - responsive viewport meta
  - title
  - body with the cursor placed inside it

Updated editor wiring:

- `src/editor/editor-input.ts` now delegates typed snippet expansion to `language-snippets.ts`.
- `src/editor/editor-keyboard.ts` checks for language snippet expansion before normal Tab indentation.
- `src/editor/CanvasEditor.tsx` accepts a `languagePath` hint separate from the real `filePath`, so snippets and syntax can use a tab name without sending fake paths to LSP or file APIs.
- `src/ui/PanelRenderer.tsx` passes `tab.path || tab.name` as the editor language hint, which lets unsaved renamed tabs like `index.html` behave like HTML for snippets.

## Session 4: Terminal Shell Recovery

### Findings

- Confirmed the issue was the terminal shell itself/render path, not the terminal tab title.
- Added a focused Rust test proving the backend PTY path can:
  - spawn the user shell
  - accept written input
  - parse emitted shell output through the `vt100` parser
- The backend test passing narrowed the live bug to shell lifetime, initial screen synchronization, and frontend rendering behavior.

### Implementation

Backend:

- `src-tauri/src/terminal/mod.rs`

  - launches common shells interactively
  - sets `SHELL`, `TERM`, `COLORTERM`, and `PWD`
  - retains the spawned PTY child handle in `PtyInstance` so the shell process stays alive
  - updates restart handling to retain the respawned child handle
  - adds `terminal_spawn_accepts_shell_input` regression coverage
- `src-tauri/src/commands/terminal.rs`
  - makes `terminal_resync` return the encoded screen delta directly while still emitting the existing event

Frontend:

- `src/ui/CanvasTerminal.tsx`

  - stops OSC title/path sequences from renaming terminal tabs
- `src/lib/actions-tabs.ts`, `src/App.tsx`, `src/lib/BusterProvider.tsx`
  - keep terminal cwd on the tab for spawn/session behavior without showing it in the tab name
