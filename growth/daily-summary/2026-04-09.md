## Session 3 — Tier 1 Editor Features (9 Items Completed)

### Smart Auto-Indent + Tab Indent/Outdent
- **Enter auto-indent** (`CanvasEditor.tsx`): pressing Enter now copies the leading whitespace from the current line to the new line
- **Tab indent** (`engine.ts:indentLines`): Tab with a selection indents all selected lines by prepending `\t`
- **Shift+Tab outdent** (`engine.ts:outdentLines`): removes leading tab or up to 2 spaces from each line in selection/current line

### Block Comment Toggle (Cmd+/)
- `engine.ts:toggleLineComment(prefix)` — toggles line comments on current line or selection
- Language-aware prefix detection in `CanvasEditor.tsx`: `//` for JS/TS/Rust/CSS, `#` for Python/Shell/YAML/TOML
- Smart behavior: finds minimum indentation across selected lines, inserts prefix there (preserves relative indentation)
- Uncomments by detecting if all non-empty lines start with the comment prefix

### Duplicate Line (Cmd+Shift+D)
- `engine.ts:duplicateLines()` — duplicates current line or all lines in selection below
- Cursor moves to the duplicated copy

### Move Line Up/Down (Alt+Up/Down)
- `engine.ts:moveLines("up"|"down")` — swaps current line (or selection range) with adjacent line
- Boundary-safe: no-op when moving first line up or last line down
- Selection follows the moved lines

### Join Lines (Cmd+J)
- `engine.ts:joinLines()` — joins current line with the next, trimming leading whitespace from the joined line
- Cursor lands at the join point

### Auto-Close Brackets and Quotes
- `CanvasEditor.tsx:handleInput()` — when typing `(`, `[`, `{`, `"`, `'`, or `` ` ``, automatically inserts the closing pair and positions cursor between them
- Smart skip: typing a closing bracket when it's already the next character skips over it instead of inserting a duplicate
- Quote-aware: only auto-closes quotes when followed by whitespace/closing bracket (prevents doubling inside words)

### Regex Search + Replace with Capture Groups
- `FindReplace.tsx:findInLines()` — now accepts `useRegex` parameter; uses `RegExp.exec()` for regex mode
- `.*` toggle button in FindReplace UI — activates/deactivates regex mode
- Invalid regex returns no matches instead of crashing
- Zero-length matches are skipped to prevent infinite loops
- `computeReplacement()` — uses `String.replace(regex, replacement)` to support `$1`, `$2` capture groups in replacement text
