# Daily Summary - 2026-04-07

## What Got Built

Terminal overhaul session. Went from a bare-bones canvas terminal to a fully interactive, Tier 3 IDE-embedded terminal with performance optimizations, user interaction features, and TUI app support.

### Terminal Performance — Delta Update System

**Problem:** Every PTY read (4KB chunk) serialized the entire cell grid (~500KB JSON for a 200x50 terminal) and shipped it over Tauri IPC. The canvas render loop redrew every frame (~60fps) even when nothing changed.

**Solution:**
- **Dirty-row delta tracking** (`compute_delta()` in Rust) — compares new screen against previous state row-by-row, only includes changed rows in the IPC payload. Typing a character now sends ~1KB instead of ~500KB.
- **Skip-frame rendering** (`needsRedraw` flag) — the `requestAnimationFrame` loop exits immediately when nothing changed. Idle terminals consume zero CPU for rendering. Cursor blink triggers redraw only on toggle (~2fps).
- `TermCell` gains `PartialEq` for efficient row comparison
- Reader thread stores `last_cells` for diffing between PTY reads

### Terminal Features — Table Stakes (Phase 1)

- Click-drag on canvas to select text across rows
- `Cmd+C` copies selected text to clipboard (falls through to Ctrl+C if no selection)

#### Mouse Reporting (Full Protocol)
- `TermScreenDelta` now carries `mouse_mode`, `mouse_encoding`, `bracketed_paste` from vt100
- Supports all mouse protocol modes: None, Press, PressRelease, ButtonMotion, AnyMotion
- Supports Default and SGR mouse encodings
- When TUI apps request mouse events (vim, htop, etc.), clicks/drags/releases forwarded as escape sequences
- When no app requests mouse events, mouse does text selection instead
- Mouse wheel sends button 64/65 when mouse reporting active

#### Scrollback UI
- Frontend maintains `scrollback: TermCell[][]` buffer (max 10K rows)
- Scroll detection before applying deltas: checks if old `cells[1]` matches new `cells[0]` (content shifted up)
- Mouse wheel scrolls through history when no TUI app has mouse reporting active
- `getVisibleRows()` composites scrollback + live cells based on scroll offset
- Scroll indicator badge ("N lines") in top-right corner when scrolled up
- Cursor hidden when viewing history; any keystroke snaps back to live view

- `\x1b[I]` sent when terminal tab becomes active
- `\x1b[O]` sent when terminal tab becomes inactive

#### Scrollback Size

## Architecture Impact

```
Before:
  PTY read → extract ALL cells → JSON serialize ALL → IPC → redraw ALL every frame

After:
  PTY read → extract ALL cells → diff against previous → JSON serialize CHANGED rows only
  → IPC → merge into persistent state → redraw ONLY when dirty
```

Interactive typing: ~500x smaller IPC payloads
Idle terminal: 60fps draw calls -> 0fps (cursor blink only ~2fps)

## Evening Session — Undo Fix + Blog Mode

### Fix: Native Menu Stealing Editor Keypresses

**Problem:** Cmd+Z (undo) did nothing in the text editor. Root cause: `PredefinedMenuItem::undo()` and `::redo()` in the Tauri native menu registered macOS system-level keyboard accelerators that intercepted keypresses before they reached the webview JavaScript. The native `undo:` Objective-C selector was sent to the hidden textarea (which had nothing to undo), so `handleKeyDown` in CanvasEditor.tsx never fired. Same issue affected Cut, Copy, Paste, and Select All.

**Fix:**
- Replaced all 6 `PredefinedMenuItem` items (undo, redo, cut, copy, paste, select_all) with custom `MenuItem::with_id` items in `lib.rs`. They still show keyboard shortcuts in the Edit menu, but fire through `on_menu_event` instead of native selectors.
- Added 6 event emitters in `on_menu_event` (`menu-undo`, `menu-redo`, `menu-cut`, `menu-copy`, `menu-paste`, `menu-select-all`).
- Added frontend listeners in `App.tsx` that route each event to the active editor engine (undo/redo/cut/copy/paste/selectAll).
- Both keyboard shortcut (via native accelerator -> event) and clicking the Edit menu item now work.

**Files:** `src-tauri/src/lib.rs`, `src/App.tsx`
