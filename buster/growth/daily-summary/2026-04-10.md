### Pretext.js Discovery

`@chenglou/pretext` is declared in `buster/package.json` but **never imported anywhere in the codebase**. Text measurement is handled by `OffscreenCanvas.measureText()` in `src/editor/text-measure.ts`. Both approaches are lightweight and avoid DOM layout, but the README claims Pretext.js specifically. This is a documentation/marketing mismatch, not a minimalism failure.

### What's Working Well (Minimalism)

- **Canvas-only rendering** — Editor, terminal, welcome screen, tour, and even the marketing website all render to Canvas 2D. Zero DOM text nodes for content.
- **SolidJS signals** — Used purely for state management and triggering canvas redraws. No component libraries, no virtual DOM.
- **Zero-dep supporting crates** — buster-path, buster-sandbox, buster-test-harness, buster-lsp-manager, buster-syntax, buster-terminal-pro, buster-remote, buster-dap all have zero or near-zero production dependencies.
- **No framework bloat** — No Tailwind, Bootstrap, Material Design, styled-components, lodash, moment, jQuery, or icon libraries.
- **11 MB installer** — Exceptional for a full-featured IDE with canvas rendering, terminal, git, AI, LSP, extensions, and syntax highlighting.

### Areas of Concern (Quality, Not Minimalism)

These don't violate minimalism but affect production readiness:

1. **DAP client UB** — `debugger/client.rs:42-46` has raw pointer cast with unsafe Send/Sync
2. **Tauri FS scope `**`** — Frontend can read/write any file on system (`capabilities/default.json`)
3. **AI command blocklist** — Substring matching in `ai/tools.rs` is trivially bypassable
4. **CRDT transform bug** — `collab/crdt.rs` Insert vs Delete returns zero instead of shift
5. **No incremental parsing** — Full tree-sitter reparse on every keystroke
6. **Full-document LSP sync** — Entire file sent on every change
7. **No `didClose` notifications** — LSP servers leak memory over time

### Key Competitive Findings

- **BUSTER is the lightest full-featured GUI IDE that exists** at 43 MB installed
- Closest Rust competitor (Zed) is 9x heavier at 391 MB
- Market is splitting into two lanes: AI-first/Electron-heavy vs performance-first/Rust-native
- BUSTER is uniquely positioned in both lanes — AI-native AND lightweight
- No other editor combines Canvas 2D rendering + Tauri + SolidJS + WASM extensions + built-in AI
- VS Code's 76% market share and 50K extension ecosystem is the biggest structural challenge
- Tauri replacing Electron is a major tailwind (104K GitHub stars, 35% YoY growth)

### Industry Trends Favoring BUSTER

1. Tauri replacing Electron
2. Rust editors gaining traction (Zed, Lapce, Helix)
3. Performance becoming a competitive differentiator again
4. Developers pushing back against bloat (Gram fork of Zed strips AI to reduce weight)
5. Privacy/local-first AI demand growing
