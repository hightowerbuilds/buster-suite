## What's Next

Items from the original critique that remain unaddressed, plus new priorities:

### Architecture

- [ ] **Replace `string[]` buffer with rope or piece table** — The critique calls this "the single most important architectural change for scalability." The `string[]` buffer works for typical files but degrades with large files (O(n) inserts, full-copy undo snapshots, no structural sharing). Mmap handler covers the read path for >1MB files but editing remains `string[]`.
- [ ] **WebGL text rendering** — Foundation built (`webgl-text.ts`) but disabled because the canvas layering broke scroll events. Needs the canvas stacking and event passthrough fixed before it can be enabled.

### Editor Features

- [ ] **Split editor (same file, two views)** — Opening the same file in two panels for side-by-side editing of the same document. Currently each tab gets its own engine instance.
- [ ] **Workspace settings** — Per-project settings file (e.g., `.buster/settings.json`) that overrides global settings. Currently only global settings exist.
- [ ] **Rainbow brackets** — Bracket matching is done but brackets aren't color-coded by nesting depth.

### AI & Intelligence

- [ ] **AI agent multi-file editing** — Agent can read/write files but lacks a multi-file edit workflow with inline diff preview before applying changes, codebase-wide context awareness, and background task execution.
- [ ] **Instant workspace indexing** — A persistent index of symbols, references, and file structure that loads instantly and enables project-wide search, rename, and navigation without per-query LSP round trips.

### Infrastructure

- [ ] **Crash reporting / telemetry** — No mechanism to learn about crashes in the wild. Needs opt-in error reporting.
- [ ] **Cross-platform testing** — Only tested on macOS. Linux and Windows builds need verification.
- [ ] **E2E tests** — Zero end-to-end tests. Need a test harness that opens the app, edits a file, saves, verifies persistence.
- [ ] **Canvas-native extension UI toolkit** — Let extension authors render custom UI on the canvas. No other editor offers this.

### Documentation & Adoption

- [ ] **User-facing documentation** — No user docs exist. Need: getting started guide, feature overview, keyboard shortcuts reference.
- [ ] **Architecture document** — Technical overview for contributors.
- [ ] **Dogfooding on diverse projects** — Only used on itself + one website. Needs testing on Python, Rust, Go, Java projects, monorepos, and small scripts.

### Foundation Deepening

- [ ] **DAP debugger UI** — Backend wired (12 commands, breakpoint state). Needs: debug panel tab with toolbar (play/pause/step), call stack view, variables view, watch expressions.
- [ ] **SSH remote UI** — Backend wired (7 commands, SFTP). Needs: connection dialog, remote file tree in sidebar, remote terminal forwarding.
- [ ] **Collab editing UI** — CRDT backend wired (9 commands). Needs: peer cursor rendering on canvas, presence indicators, connection UI, conflict resolution display.
